    Configuring kubectl / kubeconfig#

    kubeconfig location#

    If not specified by $KUBECONFIG environment variable the default kubeconfig location is $HOME/.kube/config. If $KUBECONFIG contains multiple files (similar to $PATH) then it uses an effective configuration that merges them.

    Configuration Merging#

    See https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/#merging-kubeconfig-files for details about how kubeconfig files get merged.

    File References#

    File and path references in a kubeconfig file are resolved relative to the location of the kubeconfig file. References on the command line are relative to the current working directory.

    Context#

    A kuberenetes context lets you group configuration under a convenient name. Each context is a triplet of cluster, namespace, and user.

    Show the current context#

    kubectl config current-context

    Change the current context#

    kubectl config use-context <context>

    You can also change the context for a single command by using the --context flag to kubectl,

    https://kubernetes.io/docs/concepts/configuration/organize-cluster-access-kubeconfig/ https://kubernetes.io/docs/tasks/access-application-cluster/configure-access-multiple-clusters/
