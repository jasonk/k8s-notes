    StatefulSet#

    A StatefulSet is a controller used to manage stateful applications. It manages the deployment and scaling of a set of these Pods and provides guarantees about the ordering and uniqueness of the Pods.

    A StatefulSet doesn't create a ReplicaSet controller like a Deployment does, it manages the replicas itself, giving the pods suffixed names (a deployment named db would gets pods named db-0, db-1, db-2, etc).

    Every replica of a StatefulSet will have it's own state, and each of the pods will create it's own PVC (PersistenVolumeClaim).

    apiVersion: apps/v1
    kind: StatefulSet
    metadata: { name: counter }
    spec:
    serviceName: "counter-app"
    selector: { matchLabels: { app: counter } }
    replicas: 1
    template:
    metadata: { labels: { app: counter } }
    spec:
    containers:
    - name: counter
    image: "kahootali/counter:1.0"
    volumeMounts:
    - name: counter
    mountPath: /app/
    volumeClaimTemplates:
    - metadata: { name: counter }
    spec:
    accessModes: [ "ReadWriteMany" ]
    storageClassName: efs
    resources: { requests: { storage: 50Mi } }
