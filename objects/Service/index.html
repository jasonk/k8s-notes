    Service#

    Each pod in a deployment will get created with a unique IP address, but since the pods are ephemeral these will also be ephemeral. In order to be able to access the services provided by the pod we need a Service object to provide a stable endpoint and direct traffic to the correct pods.

    In Kubernetes, a Service defines a set of Pods and a policy by which to access them. The set of Pods targeted by a Service is usually determined by a selector.

    apiVersion: v1
    kind: Service
    metadata:
    name: hello-world-service
    labels:
    app: hello-world
    spec:
    # How do we want to expose our endpoint?
    type: ClusterIP
    # How do we find pods to direct traffic to?
    selector: { app: hello-world }
    # How do clients talk to this service?
    ports: { protocol: TCP, port: 80 }
    apiVersion: v1
    kind: Service
    metadata:
    name: my-service
    spec:
    selector:
    app: MyApp
    ports:
    - protocol: TCP
    port: 80
    targetPort: 9376

    This specification creates a new Service object named "my-service", which targets TCP port 9376 on any Pod with the app=MyApp label.

    Services Without Selectors#

    A service without a selector can be used to map external resources (applications not yet moved into Kubernetes, or external databases, for example) by defining a service without a selector.

    apiVersion: v1
    kind: Service
    metadata:
    name: my-service
    spec:
    ports:
    - protocol: TCP
    port: 80
    targetPort: 9376

    Since it has no selector, the Endpoint for this Service will not be created automatically, but you can manually define endpoints.

    apiVersion: v1
    kind: Endpoints
    metadata:
    name: my-service
    subsets:
    - addresses:
    - ip: 192.0.2.42
    ports:
    - port: 9376

    Multi-Port Services#

    For some Services, you need to expose more than one port. Kubernetes lets you configure multiple port definitions on a Service object. When using multiple ports for a Service, you must give all of your ports names so that these are unambiguous.

    apiVersion: v1
    kind: Service
    metadata:
    name: my-service
    spec:
    selector:
    app: MyApp
    ports:
    - name: http
    protocol: TCP
    port: 80
    targetPort: 9376
    - name: https
    protocol: TCP
    port: 443
    targetPort: 9377
